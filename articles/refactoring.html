Refactoring Code

This is a term for resructuring code without affecting its behaviour, in order to make source more readable, consistent and maintainable. This can involve the renaming and re-arranging of code.
A number of refactoring options are available as native Visual Studio features.
  • Extract Method
  • Rename
  • Encapsulate Field
  • Extract Interface
  • Remove Parameters
  • Reorder Parameters

Renaming

To intelligently rename an object within the code. This option often appears automatically when renaming a class in Visual Studio anyway, to ensure all instances of that name are updated accordingly.

Method Extraction

Abstract Class and Interface Conversion, or Extract Interface

A new interface can be created containing members extracted from a class.
